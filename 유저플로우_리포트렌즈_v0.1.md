# 유저플로우 — 리포트렌즈 (ReportLens) v0.1

> 작성: 2026.06.22 | 기획: 장인우
> 입력: `PRD_렌즈_v0.1.md` | 범위: MVP(렌즈 2개 · 일별/월별 · 증권사 수동 업로드)
> 페르소나: 매일 리포트를 보는 PM/취업 준비자 + 주식 참고자 (본인 포함 5~10명)

---

## 1. 플로우 범위

- 시작 상태: 사용자가 오늘 받은 리포트 PDF를 손에 들고 있음
- 목표 상태: 내 목적(렌즈)에 맞게 요약된 일별 엔트리가 산업별로 누적되고, 월별 흐름을 보고 PDF로 내보냄
- 핵심 원칙: **입력은 본인 업로드만**(BYO) · **핵심숫자는 출처 인용** · **롤업은 하위 엔트리만 근거**

---

## 2. 메인 유저플로우 A — 최초 사용 (온보딩 → 첫 요약)

```mermaid
flowchart TD
    S((시작)) --> L[로그인]
    L --> O{첫 사용인가}
    O -->|예| LENS[렌즈 선택 온보딩]
    O -->|아니오| DASH[대시보드]
    LENS --> DASH
    DASH --> PICK(산업 선택 또는 새 산업 추가)
    PICK --> IND[산업 상세 화면]
    IND --> UP(리포트 PDF 업로드)
    UP --> MODAL[업로드 화면]
    MODAL --> FILE(PDF 선택 및 렌즈 확인)
    FILE --> CHK{PDF 텍스트 추출 가능}
    CHK -->|실패 스캔본| OCR[비전 OCR 경로 안내]
    OCR --> PARSE
    CHK -->|성공| PARSE(파싱 + 렌즈 적용 구조화 요약)
    PARSE --> NUM{핵심숫자 출처 확인}
    NUM -->|출처 있음| ENTRY[요약 엔트리 검토 화면]
    NUM -->|출처 불명| MARK(명시 없음 처리)
    MARK --> ENTRY
    ENTRY --> EDIT(사용자 검토 및 수정)
    EDIT --> SAVE(일별 엔트리 저장)
    SAVE --> DAY[일별 뷰에 누적 표시]
    DAY --> END((요약 완료))
```

## 3. 메인 유저플로우 B — 재방문 누적 사용

```mermaid
flowchart TD
    S((재방문)) --> DASH[대시보드: 산업 목록 + 최근 엔트리]
    DASH --> PICK(관심 산업 선택)
    PICK --> IND[산업 상세 화면]
    IND --> ACT{무엇을 할까}
    ACT -->|새 리포트| UP(PDF 업로드 후 요약)
    ACT -->|지난 흐름| TAB(시간뷰 탭 전환)
    UP --> SAVE(일별 엔트리 저장)
    SAVE --> END1((누적 완료))
    TAB --> END2((열람 완료))
```

## 4. 유저플로우 C — 시간뷰 전환 / 월별 롤업

```mermaid
flowchart TD
    S((시작)) --> IND[산업 상세 화면]
    IND --> TAB{시간뷰 탭}
    TAB -->|일별| DAY[일별 엔트리 목록]
    TAB -->|월별| MCHK{해당 월 엔트리 수}
    TAB -->|연별| YLOCK[연별 잠금: Phase 2 안내]
    MCHK -->|0개| EMPTY[엔트리 없음 안내]
    MCHK -->|1개 이상| ROLL(월별 롤업 생성)
    ROLL --> BASE(하위 일별 엔트리만 근거로 종합)
    BASE --> CONF{증권사 의견 비교}
    CONF --> MVIEW[월별 흐름 요약: 공통팩트 / 엇갈림 표시]
    MVIEW --> END((월별 열람 완료))
```

## 5. 유저플로우 D — PDF 내보내기

```mermaid
flowchart TD
    S((시작)) --> V{현재 뷰}
    V -->|일별 엔트리| E1(PDF 내보내기)
    V -->|월별 롤업| E2(PDF 내보내기)
    E1 --> PRE[PDF 미리보기]
    E2 --> PRE
    PRE --> CONF2{내용 확인}
    CONF2 -->|수정 필요| BACK(뷰로 돌아가 수정)
    CONF2 -->|확정| GEN(PDF 생성)
    GEN --> DL[다운로드 완료]
    DL --> END((내보내기 완료))
```

## 7. 의사결정 매트릭스

| 분기 조건 | 결과 A | 결과 B | 처리 방식 |
|-----------|--------|--------|-----------|
| 첫 사용 여부 | 렌즈 온보딩 | 대시보드 직행 | 온보딩 1회만 |
| PDF 텍스트 추출 | 바로 파싱 | 스캔본 → 비전 OCR 경로 | kordoc/markpdfdown 분기 |
| 핵심숫자 출처 | 인용 표기 | "명시 없음" 비움 | 가드레일: 생성 금지 |
| 월 엔트리 수 | 롤업 생성 | 엔트리 없음 안내 | 0개면 롤업 비활성 |
| 연별 탭 | (MVP 잠금) | Phase 2 안내 | 데이터 누적 후 오픈 |
| 외부 URL 입력 시도 | 차단 | BYO 업로드 유도 | 자동수집 금지 가드 |

## 8. 에러 / 예외 처리

| 케이스 | 발생 지점 | 처리 |
|--------|-----------|------|
| 지원 안 되는 파일/용량 초과 | 업로드 | 형식·용량 안내 후 재시도 |
| PDF 텍스트 추출 실패(스캔) | 파싱 | 비전 OCR 경로로 전환 안내, 실패 시 수동 입력 |
| LLM 요약 오류/타임아웃 | 요약 | 재시도 버튼 + 부분 결과 보존 |
| 요약 품질 불만 | 검토 화면 | 사용자가 직접 수정·저장(human-in-the-loop) |
| 롤업 근거 부족 | 월별 | "엔트리 N개로는 흐름 약함" 경고 표시 |
| 저작권 민감 행위(자동수집·공유) | 전역 | 기능 미제공 + 안내(개인 비공개 원칙) |
