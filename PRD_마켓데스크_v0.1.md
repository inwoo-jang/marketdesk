# PRD — 마켓데스크 (MarketDesk) v0.1 (앞단)

> 작성: 2026.06.22 | 기획: 장인우 | 갱신: 2026-06-23(빌드·프로토타입 피드백 반영)
> 위치: SKALA Plan B의 M3~4 AI 미니서비스 (실서비스 출시·운영 경험 = AI PM 1순위 우대 충족용)
> 상태: 빌드 진행 중(S0~S2). 아래는 확정 스코프.
> 갱신 요지: 취업 렌즈 **직무 15종 선택**을 MVP로 / **기업리포트·경제뉴스** MVP 포함 / **텍스트 입력** 추가(이미지 Phase2) / 산업·문서타입 **AI 자동 매칭** / **산업별 대시보드** IA.

---

## 1. 한 줄 정의

같은 산업/기업 리포트를, **사용자의 목적(렌즈)에 맞춰** 구조화 요약·인사이트로 변환하고, **일·월·연 시간 단위로 누적**해 보여주며 PDF로 내보내는 개인화 리포트 도구.

---

## 2. 배경 · 문제

- 리포트는 정보 밀도가 높지만, **읽는 목적은 사람마다 다르다**(PM/취업, 주식투자, 부동산, 산업 실무 등).
- 각자 자기 관점으로 매일 재해석하는 데 시간이 든다. 기존 요약 도구는 "객관 요약" 하나뿐이라 내 목적에 안 맞는다.
- 하루치만 봐서는 흐름이 안 보인다. **시간 축으로 누적**해야 산업 구조 변화·사이클이 보인다.

---

## 3. 타깃 & 렌즈 (MVP)

- 1차 사용자: 본인(PM/취업) + 취준 동료 + 주식 하는 지인 5~10명 (도그푸딩 = 실사용 보장)
- 렌즈(MVP 2개): `취업 인사이트`, `주식투자 참고`
- **취업 렌즈 = 직무 선택**(MVP): 기획/PM·전략/사업개발·마케팅·영업·데이터분석·리서치/애널리스트·재무/IR·컨설팅·정책/공공·법무/규제대응·구매/SCM·기자/미디어·인사/HR·개발/기술기획·기타 **15종 중 선택**. 직무에 따라 추출 관점이 달라짐.
- **멀티 렌즈**: 사용자는 렌즈를 **복수 선택** 가능(예: 취업+투자). 요약은 선택한 렌즈별 섹션으로 출력하고, 엔트리에 렌즈별로 저장 → 시간뷰·롤업도 렌즈별로 분리.
- Phase 2 렌즈: `부동산·거시`, `산업 실무 전문가`, 커스텀 렌즈

---

## 4. 핵심 가설

- (가치) 렌즈별 요약은 객관 요약보다 "내 목적에 쓸모 있다"고 느껴 **재방문(매일 사용)**한다.
- (행동) 이미 매일 리포트를 보는 사용자는 "업로드 → 렌즈 선택 → 요약"을 기꺼이 한다.
- (누적) 일별이 쌓여 월별 흐름이 보이면 사용자는 **계속 누적**할 유인이 생긴다.

---

## 5. 정보구조 (IA)

```
홈 = 내 산업 목록 (산업별 개별 대시보드)
산업 (반도체 / AI / IT·소프트웨어 / 자동차 …)   ← 표준 22 세트, 업로드 문서는 AI가 자동 매칭
 ├─ 문서 타입:  산업리포트 · 기업리포트 · 경제뉴스  (AI 자동 분류)
 ├─ 입력 형식:  PDF · 텍스트(붙여넣기)   (이미지=Phase 2)
 ├─ 렌즈:       취업(직무 선택) · 주식투자 참고  (복수 선택, 렌즈별 저장)
 └─ 시간 뷰:
     ├─ 일별:  그날 문서 요약 엔트리 (원본 분석, 출처 페이지 인용)
     ├─ 월별:  그 달 일별 엔트리 롤업 → "이 달 이 산업의 흐름"
     └─ 연별:  연간 회고 (Phase 2, 데이터 1년 누적 후)
```

화면 흐름: **홈(내 산업 목록) → 산업별 대시보드 → 일/월/연 탭.** 업로드: 문서 올리면 AI가 산업·타입 매칭(사용자 확인·수정).

추출 틀(일별 엔트리 스키마, 기존 산업리포트 스킬 계승): 신사업 / 기존사업(구조적 변화 vs 단기 변동 분리) / 해외상황 / 주식참고(가정·반대 시나리오 표시) / 핵심숫자(출처 인용) / PM 인사이트.

## 6. 핵심 기능 (MVP)

1. **문서 업로드** (PDF·텍스트) — 산업/기업 리포트·경제뉴스. 본인 보유 문서 수동 업로드(자동수집은 Phase 2).
2. **AI 산업·타입 매칭** → 업로드 문서를 표준 산업(22) 중 매칭 + 문서타입(산업/기업/뉴스) 분류. 사용자 확인·수정.
3. **렌즈 선택(복수 가능, 취업은 직무 선택)** → 렌즈/직무별로 정해진 틀로 구조화 요약 + 핵심숫자 추출(출처 페이지 인용·검증). 출력은 렌즈별 섹션.
4. **산업별 대시보드** → 산업마다 일별 엔트리 누적·열람.
5. **산업별 대시보드 / 월별 롤업** → 그 달 엔트리들을 종합한 "이 달의 흐름" (요약의 요약).
6. **PDF 정리문서 내보내기**
7. **읽기 보조**: 분석마다 **핵심 하이라이트**(가장 중요한 한 가지) 강조 + **단어 풀이**(본문 단어 클릭/검색 → AI가 100자 이내로 맥락 반영 설명, 영어 약어는 정식명 병기) + **형광펜 하이라이트**(5색, 본문 직접 칠하기·삭제, 리포트별 영구 저장). 비전문가도 산업 용어를 바로 이해.
8. **무료 한도·요금**: 기본 무료 = **하루 3회 분석**. 초과 시 → ① 유료 플랜(우리가 LLM 비용 부담, 한도 상향/무제한) 또는 ② **본인 API 키**(BYO). **BYO Claude 키 사용도 Pro(유료) 기능.** 실결제(Stripe 등) 연동은 출시 단계, 그 전까지는 사용량 미터링 + 플랜 게이팅만.

### 기본 공개 콘텐츠 (전용 스프린트, Phase 2)
- 사용자가 업로드하지 않아도 산업별로 **기본 콘텐츠**가 보이게 한다. 단 **공개·허용 소스만**(KIET·KOTRA·KIEP·정부/공공기관, 각 robots·이용조건 확인). **증권사 리포트 크롤링은 금지**(BYO 업로드 유지).
- 공용(전 사용자 공유) 콘텐츠 데이터모델 + 수집 파이프라인 필요 → 별도 스프린트. 현재 스키마는 `reports.source_type='public'` 등으로 이를 수용하게 설계.

비범위(Phase 2): **이미지(스크린샷) 입력**, **공개소스 자동수집·기본 콘텐츠**, 연별 롤업, 부동산·전문가·커스텀 렌즈, 주식 시그널, 노션 자동연동, 실결제 연동.

## 7. 시간 뷰 롤업 설계 (AI 핵심 포인트)

- 월별·연별은 원문 재독이 아니라 **이미 만든 일별 엔트리를 롤업**하는 계층적 요약.
- 상위 요약은 **하위 엔트리만 근거**로 삼는다 → 새 숫자·전망 생성 금지(계층별 환각 누적 차단).
- 여러 증권사 의견이 겹치면 **"공통 팩트"**, 엇갈리면 **"엇갈림"**으로 표시.

## 9. 가드레일 정책 (이 제품의 생명)

**(A) 콘텐츠 품질**
- 리포트에 없는 숫자·전망 **생성 금지** → 모든 핵심숫자에 출처 페이지 인용 강제
- 불확실하면 "리포트에 명시 없음"으로 비움(지어내지 않음)
- 롤업(월/연)은 하위 엔트리만 근거, 새 사실 생성 금지
- 투자 렌즈: **투자조언 아님 + 참고용** 면책, 국내 매도의견 드문 점 보수적 해석

**(B) 저작권 · 데이터**
- 입력은 **사용자가 직접 업로드한 본인 보유 문서만** (서비스가 외부에서 자동 수집하지 않음)
- 출력은 **변형적 요약**(원문 표현 장문 복제 금지, 핵심숫자만 출처 표기로 인용)
- 저작물 처리 데이터는 **사용자 본인에게만** 노출 (타 사용자 공유·공용 피드 차단)

## 10. 운영 · 법적 리스크 (식별 & 대응)

- **리스크:** 증권사 리포트는 저작권상 "자동 대량 수집·무단 전재·상업적 재배포" 금지. 네이버 금융 리서치는 **robots.txt(제3자 봇 /research/ 차단, yeti만 허용) + 이용약관(자동화 수단 수집 금지)** 으로 스크래핑 불가 확인(2026.06).
- **대응:** 수집형 → **BYO(지참형)** 으로 설계. 증권사 = 본인 수동 업로드. 자동화가 필요하면 **공개·허용 소스**로 한정.
  - 합법 자동화 후보: 산업연구원(KIET), KOTRA 해외시장뉴스, KIEP, 정부·공공기관 보고서 (각 사이트 robots·이용조건 개별 확인).
- 이 "수집형 대신 BYO·비공개·변형적 출력"으로 리스크를 설계로 회피한 것 자체가 면접용 기획 서사.

---

## 11. 규칙 vs AI 경계

- 규칙/검증: 숫자·날짜·기관명 추출과 출처 매칭, 틀 항목 채움 여부 체크, 롤업 시 근거 엔트리 제한
- AI: 렌즈별 인사이트·해석 생성, 월별 흐름 종합
