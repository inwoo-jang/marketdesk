# 마켓데스크 (MarketDesk)

산업·기업 리포트와 경제뉴스는 매일 쌓이는데, 읽는 목적은 사람마다 다릅니다. 마켓데스크는 내가 올린 문서를 취업(직무별)·투자 같은 내 관점으로 정리해 주고, 산업별로 모아 흐름까지 짚어 줍니다.

> 혼자 하는 사이드 프로젝트 (장인우 · Plan B). 프로토타입이 아니라 상용화 가능한 구조가 목표.
> 문서 갱신: 2026-07-07

- ✅ **Sprint 2 (AI 코어)**: PDF·텍스트 → 산업·타입 AI 분류 → 렌즈별 추출 → 출처 가드레일 → 검토 화면. 산업별 대시보드·표준 산업세트·취업 직무 반영.
- 🔵 **Sprint 3 (흐름·정리·읽기)**: **흐름 보드**(월/년 × 산업/기업/경제흐름, 산업 관점 종합 요약, 칸별 재생성) · **공공 콘텐츠**(정책브리핑 API) · **계열/기업 별표** · **서식 메모** · **업로드 중복 감지** · **화면별 사용법 가이드**(플로팅 ?) · PDF 내보내기까지 로컬 풀스택 동작.

> 개발은 **로컬 우선**. 로컬 Postgres + 로컬 LLM(Claude/Codex CLI) 또는 Gemini 로 끝까지 돌아가고, AWS 는 출시 단계에 붙입니다.

## 무엇을 다루나

- **입력 형식**: PDF · 텍스트(붙여넣기). *이미지(스크린샷)는 다음 단계.*
- **문서 타입**: 산업리포트 · 기업리포트 · 경제뉴스 — 업로드 시 **AI 가 자동 분류**.
- **산업**: 표준 24개 세트(반도체·AI·IT·소프트웨어·자동차·석유화학·환경·기후·금융 등). 업로드 문서의 산업을 **AI 가 자동 매칭**(사용자 확인·수정 가능).
- **렌즈(목적)**: 취업(직무 15종 중 선택) · 투자. 복수 선택 가능, 렌즈별로 분리 저장.
- **추출 틀**(문서타입 공통): ①한줄요약 ②핵심사실 ③동인·맥락 ④리스크·쟁점 ⑤관점레이어(투자/취업) ⑥핵심숫자(출처 `[p.N]`). 리포트당 1분석.
- **렌즈 편집·재분석**: 원문(PDF·텍스트)을 서버에 보존 → 취업/투자 렌즈·직무를 바꿔 **다시 요약**. 재분석도 분석 1건으로 게이팅.
- **읽기 보조**: 핵심 하이라이트 + 단어 풀이(본문 단어 클릭/검색 → AI 100자 설명, 맥락 반영, 약어 병기) + 형광펜(5색) + **앵커 메모**(본문 선택 → 우측 여백 노트, 스크롤 동행) + 영문 기사는 한국어 요약.
- **서식 메모**: 흐름 상세·리포트에 자유 메모 — **손글씨 3종·H1~H3·굵기·글자 크기·형광펜 5색·접기**, 보기/편집 분리, **PDF 내보내기에 포함**.
- **정리·탐색**: 리포트·공공 **책갈피·숨김** + 탭(전체/즐겨찾기/숨김) + 페이지네이션 + **docType·기간 필터·정렬(발간일순/최근 3개월 업로드순)**. 문서 타입별 카드 색 구분(산업 보라·기업 앰버·뉴스 하늘·공공 초록).
- **별표(즐겨찾기)**: 관심 산업 + **계열/기업 별표**(공정위 계열사 매핑, 국내/해외 기타 분리), 대시보드·흐름보드에 반영, **드래그로 순서 변경**.
- **흐름 보드**: 월/년 × 산업(★관심·기타 맨밑)·기업(계열별/산업별)·경제흐름 타임라인. **월별은 연도 선택**. 셀 → **흐름 피드(산업 관점 종합 요약 + 산업/기업/경제흐름 구분 원문)** → 원문 드릴다운. 업로드 시 자동 갱신, **칸별/부분 재생성**, 재생성 중에도 이전 내용 유지(실패는 물음표만).
- **흐름 요약 품질**: 그 산업으로 태그된 **모든 리포트+공공**을 근거로(대시보드와 정합), 기사별 나열이 아니라 **산업 관점으로 종합**(노이즈 제거, 핵심 3~5개), 리포트 메타 표현 금지.
- **업로드 중복 감지**: **정확 중복**(내용 SHA-256 → 업로드 차단·안내), **유사 중복**(본문 SimHash → 배너·병합).
- **가드레일**: 리포트에 없는 숫자 생성 금지, 핵심숫자 출처 페이지 룰매칭 검증, 투자 면책.
- **요금**: 무료 = 하루 3회 분석. 초과 시 유료 플랜 또는 본인 API 키(BYO). 실결제 연동은 출시 단계.

## 사용법

앱 오른쪽 상단의 플로팅 **?** 버튼을 누르면 **지금 화면에서 할 일**을 팝오버로 알려준다(?·바깥·화면 이동 시 닫힘). 팝오버의 "전체 사용법 보기"(`/guide`)에서 아래 전체 흐름을 화면으로 볼 수 있다.

1. **렌즈 고르기** — 취업(산업 구조·기업·채용 시그널) 또는 투자(실적·수급·전망 숫자) 관점을 켠다. 환경설정에서 언제든 변경, 취업 렌즈는 직무까지 지정.
2. **리포트 올리기** — PDF·텍스트 업로드. 산업은 비우면 AI 가 자동 분류하고, 켠 렌즈로 구조화 요약. 중복(정확·유사) 자동 감지. 원문은 보존돼 렌즈를 바꿔 다시 요약 가능.
3. **리포트 찾아보기** — 산업/기업/뉴스 메뉴. 기업은 계열별·산업별 필터 + 별표 고정. 카드에서 요약·원문 확인.
4. **대시보드** — 관심 산업 + 별표 기업의 월별 흐름을 모아 봄. 산업 클릭 시 그 산업의 월별 흐름과 근거 원문.
5. **흐름 보드** — 산업별·기업별·경제흐름을 월/년 타임라인으로. 빈 칸 생성, 칸별 ↻ 재요약, 셀 클릭 → 흐름 피드 + 근거 원문.

그 밖에 공개 자료 자동 수집(대시보드 공공 탭), 용어 클릭 풀이, 서식 메모(PDF 내보내기), 저장(북마크)을 지원한다.

## 로컬 실행

사전: PostgreSQL 16 (로컬 실행 중), Node 20+, pnpm 9.

```
pnpm install
pnpm db:migrate        # 스키마 생성
pnpm db:seed           # 렌즈·산업 시드
pnpm dev               # api + web + worker 동시 실행
```
→ 브라우저 http://localhost:3000 (dev 로그인 → 온보딩 → 대시보드 → 업로드 → 검토)

> `.env` 는 각 패키지의 `.env.example` 참고해서 생성(`api/.env`, `web/.env.local`, `worker/.env`, `packages/db/.env`). 실제 LLM 은 `worker/.env` 의 `LLM_PROVIDER=gemini` + `GEMINI_API_KEY`, 또는 개발자 설정 화면에서 로컬 Claude/Codex CLI를 선택해 사용.

## 데이터 포함해서 넘기기

`git clone` 은 코드만 받는다. 분석된 리포트 데이터는 로컬 Postgres, 업로드 원본은 `api/.uploads/` 에 있고 둘 다 git 추적 대상이 아니다. 데이터까지 그대로 넘기려면 번들 파일을 따로 전달한다(구글드라이브 등). 시크릿(.env)은 포함하지 않으니 받는 쪽이 본인 키로 `.env` 만 새로 만든다.

넘기는 쪽:

```
pnpm data:dump         # DB 덤프 + 업로드 원본 → marketdesk-data-bundle.tgz
```

받는 쪽(코드 clone + `.env` 작성 후):

```
pnpm install
pnpm db:migrate                          # 스키마 생성
pnpm data:restore <받은 번들 경로>.tgz   # 리포트 데이터 + 원본 복원
pnpm dev
```

> 연결정보는 `DATABASE_URL`(없으면 `postgres://<사용자>@localhost:5432/reportlens_dev`). 번들 파일(`*-data-bundle.tgz`)은 `.gitignore` 로 커밋 차단돼 있다. 기존 분석 데이터·원문은 키 없이도 조회 가능하고, 새 업로드나 흐름 생성 때만 LLM 키가 필요하다.
